# UnoDOS/C64 — standalone OS for the Commodore 64 (milestone 1)

Like the other bare-metal UnoDOS ports, this is **a real operating system**,
not a BASIC program that paints a screen. A C64 powers on into BASIC; you
`LOAD"UNODOS",8,1` and `RUN`, the one-line BASIC stub does `SYS 2061`, and from
there UnoDOS **owns the machine** — it banks the VIC-II into a hi-res bitmap,
runs its own renderer, window manager and desktop, scans the keyboard matrix
itself, reads the clock straight off the CIA, and never calls the KERNAL or
returns to BASIC.

## What the C64 brings over the 1-bit ports

The Apple II port is "UnoDOS Lite" because a 1 MHz 6502 with a software 1-bit
hi-res renderer is the tightest envelope in the family. The C64 shares the CPU
(the 6510 is a 6502 with an on-chip banking port) but the *machine* is richer,
and M1 leans into that rather than re-deriving the Apple II:

- **Colour, not 1-bit.** Standard VIC-II hi-res bitmap gives **two colours per
  8×8 cell** from screen RAM (upper nibble = the "1"/foreground pixel colour,
  lower nibble = the "0"/background). So the desktop, menu bar, windows, title
  bars and icons are genuinely 16-colour: blue dithered desktop, light-grey
  menu bar, white windows with **blue focused / grey unfocused** title bars,
  blue-highlight selected icons. The bitmap holds the *shapes* (borders, text,
  dither); screen RAM holds the *colours*.
- **A real hardware clock.** The Clock app reads the **CIA #1 Time-of-Day**
  registers (`$DC08`–`$DC0B`, BCD), with the hardware read-latch the chip
  requires (reading hours latches the whole time; reading tenths releases it).
  No calibrated soft tick like the Apple II — this is an actual clock that ran
  off the mains frequency.
- **PAL/NTSC auto-detection.** SysInfo samples the raster counter
  (`$D012` + `$D011` bit 8) and keeps the maximum 9-bit line number: PAL tops
  out at line 311 (`$137`), NTSC at 262 (`$106`). It reports the right CPU
  clock (0.985 vs 1.023 MHz) and VIC-II part (6569 vs 6567).
- **SID.** App launch plays a short blip on SID voice 1 (the harness counts
  register writes so a test can assert it). M2/M3 will use the three voices for
  Music/Tracker.

## Memory map (VIC bank 1)

The kernel keeps its code/data low and puts everything the VIC reads in bank 1
(`$4000`–`$7FFF`, selected via CIA #2 `$DD00`):

| Region | Use |
|---|---|
| `$0801`–`$3FFF` | BASIC stub + kernel code/data (~14 KB; M1 uses ~3 KB) |
| `$4000`–`$43E7` | screen RAM — per-cell `fg<<4 | bg` colour nibbles |
| `$6000`–`$7F3F` | hi-res bitmap (8000 bytes, 320×200) |
| `$C000`–`$CFFF` | free RAM reserved for M2/M3 buffers |

`$D018 = $08` puts the screen at bank+`$0000` and the bitmap at bank+`$2000`;
`$D011 = $3B` selects hi-res bitmap mode. The byte holding pixel *(x, y)* is at
`$6000 + (y>>3)*320 + (x>>3)*8 + (y&7)` (cells 8 bytes apart, the 8 rows within
a cell consecutive) with **bit 7 = leftmost pixel** — which is exactly the
shared UnoDOS font convention, so the 8×8 glyphs are used byte-for-byte (no
re-packing, unlike the Apple II's 7px layout). The address tables
(`build/tables.s`) and font (`build/font8.s`) are generated by `mktables.py` /
`mkfont.py`.

## Boot chain

1. BASIC loads the PRG at `$0801`; `RUN` interprets the stub (`10 SYS 2061`).
2. `start` (`$080D`): `SEI`, reset the stack, switch the VIC to bank 1 + hi-res
   bitmap, point `$D018`/`$D011`/`$D016` at our screen+bitmap, set the CIA #1
   keyboard port directions, seed the TOD clock if it isn't running, and
   **auto-detect PAL/NTSC**.
3. Clear the bitmap, draw the desktop, (AUTOTEST: open SysInfo + Clock), draw
   the icons, and fall into the cooperative `mainloop` — poll the keyboard
   matrix (edge-detected) and refresh the Clock when the CIA second ticks.

## M1 desktop

40×25 cells of the hi-res bitmap:

| Cells | Content |
|---|---|
| row 0 | menu bar (`UnoDOS/C64`, black on light-grey) |
| rows 2–9 | **SysInfo** window — machine / CPU / RAM / video / SID |
| rows 11–15 | **Clock** window — `HH:MM:SS` from the CIA TOD |
| rows 1–24 | blue dithered desktop behind the windows |
| rows 20–22 | icon grid — SysInfo, Clock |

**Window manager:** a z-order list (`zlist`, `[0]` = topmost/focused) with two
fixed-position windows; open/raise to the front, **RUN/STOP** (ESC) closes the
topmost, the new top takes focus (its title bar turns blue). **Keyboard:** the
CIA #1 matrix is scanned each loop into a single decoded key with edge
detection — `CRSR ⇄` (with SHIFT for left) moves the icon selection, `RETURN`
opens/raises the selected window (with a SID blip), `RUN/STOP` closes.

## Building

```sh
./build.sh        # build/unodos_c64.prg + .d64           (plain desktop)
./build.sh test   # build/unodos_c64_test.prg + .d64      (AUTOTEST: SysInfo + Clock open)
```

Needs `dasm` (the 6502/6510 cross-assembler — `DASM` env var or
`C:\Users\arin\apple2-tools\dasm.exe`) and Python 3. `mktables.py` emits the
VIC bitmap address tables, `mkfont.py` copies the shared 8×8 font verbatim,
`dasm` assembles `kernel.s` (`org $0801`), and `mkprg.py` prepends the load
address to make the `.prg` and packs it into a 35-track 1541 `.d64` (real BAM +
directory + interleaved file chain) named `UNODOS`.

## Testing without real hardware

A cycle/timing-accurate C64 (VICE) is overkill for regression screenshots, so
this port ships its own ROM-free harness: [harness.py](harness.py) is a `py65`
6510 core that plays the part of the machine around the kernel —

- loads the PRG and jumps straight to `start` (the BASIC interpreter is for
  real hardware);
- advances the **VIC raster** (`$D012` + `$D011` bit 8) with CPU steps so the
  kernel's PAL/NTSC detector sees a full line sweep (default PAL; `--ntsc`);
- serves the **CIA #1 keyboard matrix** ($DC00 column-select → $DC01 rows) from
  whatever keys a `key`/`keys` command is holding;
- presents the **CIA #1 Time-of-Day** clock (base 12:00:00) advancing from CPU
  steps, with the hours-latch / tenths-release behaviour the kernel relies on;
- counts **SID** register writes (so `assert beep>0` works);
- renders the hi-res bitmap + per-cell screen-RAM colour through the 16-colour
  C64 palette to an **RGB PNG**.

```sh
./build.sh test
python3 harness.py build/unodos_c64_test.prg shots < tests/m1.script
```

[tests/m1.script](tests/m1.script) drives the M1 surface: boot to the AUTOTEST
desktop (SysInfo + Clock open), assert PAL was detected, move the icon
selection, raise SysInfo (assert the SID blipped), close it with RUN/STOP, and
let the CIA clock advance — 5 screenshots (`m1_boot`, `m1_select`, `m1_launch`,
`m1_close`, `m1_clock`). [tests/m1_ntsc.script](tests/m1_ntsc.script) re-runs
boot with `--ntsc` and asserts the detector flips to NTSC.

Two calibration constants matter (see harness.py): `TICK_INSTRS` (7000)
instruction-steps per `wait` tick — the full-screen dither + window draws make
boot cost ~360k steps, so a leading `wait` clears it — and `STEPS_PER_SEC`
(30000), which ties the CIA TOD clock to CPU steps so `wait` advances HH:MM:SS
predictably.

## Real-hardware path

`build/unodos_c64.prg` runs in **VICE** (`x64sc`) via *Smart attach* or
`LOAD"UNODOS",8,1 : RUN` from the `.d64`. The same `.d64` goes on an **SD2IEC**
or **1541 Ultimate** for a real C64. M1 is harness-verified only; before metal,
boot it in VICE to confirm the VIC bank switch, the keyboard matrix decode and
the CIA TOD all behave on a cycle-accurate machine. The kernel takes the whole
machine with `SEI` + polling and makes **no KERNAL calls**, so it does not
depend on a particular KERNAL revision — but it does assume the standard CIA #1
keyboard wiring and a running TOD (it seeds 12:00:00 if the clock reads zero).

## Milestones

- **M1 (this):** boot chain (BASIC stub → VIC bitmap takeover), colour hi-res
  desktop + menu bar, keyboard-driven window manager (SysInfo + Clock),
  PAL/NTSC detection, CIA Time-of-Day clock, CIA keyboard-matrix scanner, SID
  blip, shared 8×8 font, ROM-free py65 harness + `tests/m1.script`.
- **M2 (planned):** a 1541 storage path (IEC or a `.d64` block driver), a
  mini-FS, Files + Notepad, letter/digit keyboard decode.
- **M3 (planned):** the scaled app roster (Theme — trivially colourful on the
  VIC, Dostris, Pac-Man, Music + Tracker on the three SID voices, Paint) and
  the OutLast / scheduler feasibility verdicts, as on the other ports.
